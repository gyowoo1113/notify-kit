### 프로젝트 한 줄 소개
- 멀티테넌시 기반의 알림 도메인을 “생성-저장-전송-재시도-읽음처리”까지 일관되게 제공하는 백엔드 중심 오픈소스형 모듈

### 주요목표
- 여러 프로젝트에서 라이브러리 형식으로 활용가능한 알림 오픈소스 서비스 구현
- 헥사고날 아키텍쳐 기반으로 JPA/Mybatis 등 구현체 교체(커스텀) 기능 제공
- 주요 구조가 Spring에 종속적이지 않도록 core/spring 모듈 분리
- 점진적으로 기능 확장 및 성능 테스트 적용
- 모바일 환경에서도 사용가능하도록 적용

### 고민점
- 구현방법 선택 : Short Polling / Long Polling / Web Socket / Server Sent Events
- 목표: 기능의 모듈화 => (외부)프로젝트에서 알림 기능 사용을 위해 필요한 추가 설정이 최소화 되어야 함

### 주요 기능(What) : 목표
- 멀티테넌시(tenantId/projectKey) 기반 알림 격리
- Idempotency Key 기반 중복 알림 방지
- Delivery Log(성공/실패/재시도/에러) 기록으로 운영 가시성 확보
- Read/Unread 모델 및 효율적인 조회 인덱싱
- WebSocket 실시간 전송 + 확장 가능한 채널 추상화(SSE/FCM)
