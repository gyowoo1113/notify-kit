### 프로젝트 한 줄 소개
- 멀티테넌시 기반의 알림 도메인을 “생성-저장-전송-재시도-읽음처리”까지 일관되게 제공하는 백엔드 중심 오픈소스형 모듈

### 문제 정의(Why)

- 웹소켓 기반 실시간 알림은 단일 서버에서는 간단하지만, 서버가 수평 확장될 때 “어느 서버에 어떤 사용자가 연결되어 있는지”를 모르면 안정적인 실시간 전달이 깨진다.
- 또한 알림은 사용자 경험과 운영 품질에 직결되므로, 단순 푸시가 아니라 중복 방지, 전송 실패 재시도, 전달 로그, 읽음 모델 등 도메인 전반을 설계해야 한다.

### 해결 방식(How)
- Core/Adapter 구조로 설계하여 기본 모드(InMemory)에서는 외부 의존성 없이 동작하고, 운영 모드에서는 Redis EventBus 및 RDB Store를 플러그인으로 장착할 수 있도록 설계
- WS 서버 다중 인스턴스 환경에서 Redis pub/sub 기반 이벤트 브로드캐스팅으로 “연결 서버를 모르는 문제”를 해결하고, 각 서버가 자신이 관리하는 세션에만 전달하도록 구성
- 알림 저장과 이벤트 발행의 정합성을 위해 Outbox 패턴을 적용하여 장애 상황에서도 유실/중복 가능성을 낮춤

### 주요 기능(What) : 목표
- 멀티테넌시(tenantId/projectKey) 기반 알림 격리
- Idempotency Key 기반 중복 알림 방지
- Delivery Log(성공/실패/재시도/에러) 기록으로 운영 가시성 확보
- Read/Unread 모델 및 효율적인 조회 인덱싱
- WebSocket 실시간 전송 + 확장 가능한 채널 추상화(SSE/FCM)
