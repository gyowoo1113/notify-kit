## Why SSE is Optional

실시간 알림 전송은 유용하지만, 모든 프로젝트에서 항상 필요한 기능은 아님
notify-kit은 라이브러리 형태로 제공되므로, SSE를 **옵션 기능**으로 분리함

### 설계 목적

- 기본 CRUD 사용 시 불필요한 실시간 전송 의존성을 강제하지 않음
- 전송 방식(SSE, WebSocket, FCM 등)을 환경에 따라 선택 가능하게 함
- 전송 실패가 비즈니스 로직에 영향을 주지 않도록 분리함

### 적용 방식

- 기본 상태에서는 `NoopNotificationEventPublisher`가 주입되어 전송을 수행하지 않음
- `notify.sse.enabled=true` 설정 시 SSE 기반 구현체가 활성화됨

### 설계 효과

- 기본 기능은 가볍게 유지하고, 필요할 때만 실시간 기능을 활성화할 수 있음
- 라이브러리 사용자에게 명확한 선택 지점을 제공함
