## Why `NotificationFacade`

`NotificationFacade`는 notify-kit을 사용하는 외부 프로젝트에서의 **단일 진입점** 역할을 함

### 설계 목적

- core 유스케이스(`NotificationService`)를 외부에 직접 노출하지 않음
- 트랜잭션 경계, 이벤트 발행 등 애플리케이션 흐름 제어를 한 곳에 모음
- 저장, 이벤트 발행, 옵션 기능(SSE 등)을 조합하는 역할을 담당함

### 설계 효과

- 외부 프로젝트에서는 Facade만 호출하면 되도록 단순화됨
- core 모듈의 구조 변경이 외부 사용자에게 직접적인 영향을 주지 않음
- 애플리케이션 레벨의 오케스트레이션 로직을 명확히 분리할 수 있음
